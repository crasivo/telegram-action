## Как сообщить об уязвимости

**Пожалуйста, не используйте публичные GitHub Issues для сообщений об уязвимостях.**

Если вы обнаружили потенциальную проблему безопасности в этом проекте, пожалуйста, сообщите об этом ответственно:

1. **Отчет:** Отправьте письмо на адрес **[crasivodev@gmail.com]** или воспользуйтесь функцией частных отчетов об уязвимостях в
   настройках GitHub.
2. **Детали:** Опишите уязвимость, шаги для её воспроизведения и, если возможно, приложите пример (proof of concept).
3. **Ответ:** Вы получите подтверждение о получении вашего сообщения в течение 72 часов.

## Рекомендации для пользователей

Чтобы ваши данные оставались в безопасности:

- **Используйте Secrets:** Никогда не записывайте `bot_token` прямо в код workflow. Всегда
  используйте [GitHub Secrets](https://docs.github.com/en/actions/security-guides/using-secrets-in-github-actions).
- **Проверка:** Убедитесь, что вы используете официальную версию экшена из GitHub Marketplace.

Спасибо за помощь в обеспечении безопасности проекта!
